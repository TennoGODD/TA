# Task Advancer

Личный QA-инструмент: автоматически продвигает производственные задания DMC
по статусам жизненного цикла до выбранной цели — без ручного прохождения
через Manager / Monitor / Print Station и без реального оборудования.

Третье приложение в наборе (вместе с **KMTasker** — создание заданий и
**LineEmulator** — эмуляция оборудования линии), в том же стиле.

## Что делает

Принимает `taskid` уже созданного задания (статус **3** «готово к печати»
или **6** «готово к сериализации») и доводит его до одной из целей:

| Цель | Статус | Что происходит по пути |
|------|--------|------------------------|
| Отчёт о нанесении КМ | 9 | Верификация всех КМ и агрегатов, сборка КИТУ/паллет, перевод 6→8, сервер сам 8→9 |
| Ввод в оборот | 12 | + PATCH 10 → сервер отправляет отчёт о нанесении в ЧЗ → сам 10→11→12 |
| Завершено | 99 | + PATCH 13 → сервер отправляет ввод в оборот → сам 13→…→99 |

⚠️ Цели 12 и 99 запускают на сервере **реальную отправку документов в
«Честный знак»** — использовать только на стенде с демо-контуром ЧЗ.

## Как это работает

- **Чтения и верификация** — напрямую в PostgreSQL DMC (`UPDATE dm/aggregates SET status=30`).
- **Мутации** — через официальный REST API v2 (`PATCH /api/v2/tasks/{id}`,
  `POST /api/v2/build_aggregate/`) — тем же путём, что и настоящий Monitor.
- **КИГУ** (уровень 0) предзарезервированы сервером — только верифицируются.
- **КИТУ/паллеты** собираются **снизу вверх** (build_aggregate требует, чтобы
  дочерние коды уже существовали в БД); код агрегата генерируется локально по
  формуле штатного «Стандартного генератора DMC».
- Каждое задание — отдельный фоновый поток; ошибки одного не влияют на другие.

## Запуск

```
pip install -r requirements.txt
python src/main.py
```

Сборка exe: `pyinstaller --noconfirm TaskAdvancer.spec`

## Настройки

⚙️ в сайдбаре: PostgreSQL DMC (host/port/dbname/user/password), REST API v2
(host/port, по умолчанию 8025), интервал опроса статуса, звук уведомлений.
Хранятся в `db_config.json` / `api_config.json` / `app_config.json` рядом с exe.
Список заданий — `tasks.json`. Лог — `logs/taskadvancer_log.txt`.
