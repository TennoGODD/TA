# Task Advancer — история версий

## 1.1 — 2026-07-20

- **Контроль серверных ошибок (`error_state`)**: если сервер DMC пометил
  задание ошибкой, продвижение немедленно завершается с текстом ошибки
  (включая сообщение сервера из поля `msg`). Проверка выполняется при
  добавлении, на каждом опросе статуса и во время ожидания готовности.

## 1.0 — 2026-07-17 (первый релиз)

Личный QA-инструмент: автоматически продвигает производственные задания DMC
по статусам жизненного цикла до выбранной цели — без ручного прохождения
через Manager / Monitor / Print Station и без реального оборудования.
Третье приложение линейки (вместе с KMTasker и LineEmulator), в том же стиле.

### Возможности

- **Продвижение задания до цели**: «Требуется отправить отчёт о нанесении КМ (9)»,
  «Можно передать данные для ввода в оборот (12)» или «Завершено (99)».
  По пути автоматически: верификация всех КМ, верификация/сборка агрегатов,
  переводы статусов 3 → 6 → 8 → 9 → 10 → … с ожиданием реакции сервера DMC.
- **Агрегация любых уровней**: предзарезервированные КИГУ верифицируются и
  наполняются КМ (привязка `dm.aggregate`), КИТУ и паллеты собираются через
  `build_aggregate` строго снизу вверх — как это делает настоящий Monitor.
  Коды агрегатов генерируются по формуле штатного «Стандартного генератора DMC».
- **Ожидание готовности**: задание в подготовительном статусе (0/1/2) можно
  добавить в список с кнопкой «⏳ Ждать готовности» — программа дождётся
  статуса 3/6 и запустит продвижение автоматически.
- **Автоподсказка номера задания** — выпадающий список заданий в статусах
  3/6 прямо из БД, вставка из буфера работает на любой раскладке.
- **Параллельная работа**: каждое задание идёт в своём фоновом потоке,
  ошибка одного не влияет на остальные; для упавшего задания — кнопка
  «🔁 Повторить» и просмотр текста ошибки.
- **Защита от залипаний**: если задание застряло в статусе 9 после перевода
  в 10 — перевод повторяется автоматически; если сервер с автоотправкой
  проскочил целевой статус — это распознаётся и помечается как успех.
- **Уведомления**: звук, всплывающий тост и мигающая зелёная рамка карточки
  при достижении цели (рамка остаётся зелёной).
- **Настройки**: подключение к PostgreSQL DMC и REST API v2, интервал
  опроса, звук. Список заданий переживает перезапуск (`tasks.json`),
  подробный лог — `logs/taskadvancer_log.txt`.
